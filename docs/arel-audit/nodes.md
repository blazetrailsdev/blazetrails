# nodes/ — file-by-file audit

Source: `activerecord-8.0.2/lib/arel/nodes/*.rb` vs `packages/arel/src/nodes/*.ts`.

## node.rb → node.ts — **OK with EXTRAS**

- Methods present: `not`, `or`, `and`, `invert`, `toSql` (Rails `to_sql`), `fetchAttribute`, `isEquality` (Rails `equality?`).
- Rails: includes `Arel::FactoryMethods`. TS: declared via `interface Node extends FactoryMethodsModule`, runtime wired in `index.ts`. Match.
- EXTRA: `eql`, `hash` are inherited from base/built in Ruby; TS implements via stable serialization. `setToSqlVisitor`, `registerNodeDeps` are TS-only registry hooks needed because of ESM cycle constraints. Documented inline.
- DRIFT: Method renames `to_sql → toSql`, `equality? → isEquality`. api:compare needs the rename map.

## binary.rb → binary.ts — **OK with EXTRAS**

- Binary base class present with `left`, `right`, `eql`, `hash`. ✓
- Subclasses generated by `const_set` in Rails: `Assignment, Join, Union, UnionAll, Intersect, Except`. TS has all + extends `Join` to abstract for type safety, plus `CrossJoin extends Join`.
- Named subclasses with `invert`/FetchAttribute: `As, Between, GreaterThan, GreaterThanOrEqual, LessThan, LessThanOrEqual, IsDistinctFrom, IsNotDistinctFrom, NotEqual, NotIn`. All present and `invert` returns expected counterpart. ✓
- DRIFT: `As#to_cte` in Rails uses `Arel::Nodes::Cte.new(left.name, right)` — uses `left.name` (calls `.name` on left). TS uses `right` for the name and `left` for the relation; recently corrected via the `_registerCteFactory` registry. Behavior **looks swapped** vs Rails — Rails `As(left, right)` where left is the relation node with `.name`, right is the alias literal: Cte built as `(left.name, right)`. Our `toCte()` calls `cteFactory(name, this.left)` with name pulled from `this.right` (the alias literal), and relation from `this.left`. Reading Rails again: `Cte.new(left.name, right)` — name = `left.name`, relation = `right`. **GAP**: TS swaps these — confirm with a unit test. (See `binary.ts:80-91`.)
- EXTRA: Trails `Binary` subclass also exposes `as`, `and`, `or`, `not` instance methods as conveniences. Rails has these via the `NodeExpression` mixin chain (Binary → NodeExpression → AliasPredication etc). Match in spirit.
- EXTRA: `fetchAttributeFromBinary` helper, `ATTRIBUTE_BRAND` symbol — TS-only mechanism replacing Ruby `is_a? Arel::Attributes::Attribute`. Documented.

## unary.rb → unary.ts — **OK with one DRIFT**

- Rails `const_set` list: `Bin, Cube, DistinctOn, Group, GroupingElement, GroupingSet, Lateral, Limit, Lock, Not, Offset, On, OptimizerHints, RollUp`.
- TS has all 14, plus extras `Top` (not in Rails — verify if needed; api:compare will flag).
- DRIFT: TS `Not` extends `Node` (not `Unary`); TS `Lateral` extends `Node`; TS `GroupingElement`/`Cube`/`RollUp`/`GroupingSet` extend `Node`. Rails has them all extending `Unary`. **GAP**: Subclass hierarchy diverges — visitor dispatch may treat `Lateral` / `Not` / `GroupingElement` differently from other `Unary` subclasses.
- DRIFT: `OptimizerHints` carries a typed `hints: ReadonlyArray<string|SqlLiteral>` field instead of using the inherited `expr` slot. Rails uses `expr`. Documented.
- DRIFT: TS `Unary#expr` is `unknown`; Rails is implicit Ruby. PR 31/32 widening preserves backward compat. ✓
- EXTRA: Backwards-compat alias `Rollup = RollUp` (deprecated). Recently added.

## nary.rb → nary.ts — **DRIFT**

- Rails: `Nary` base + `And = Class.new(Nary)`, `Or = Class.new(Nary)`. Both live in nary.rb.
- TS: `nary.ts` defines a base; **`And` and `Or` live in separate files** (`and.ts`, `or.ts`). EXTRA files; api:compare needs to redirect `And`/`Or` to `nary.rb` — see extras.md.
- GAP: Rails `Nary#fetchAttribute` walks all children with `all?`. Verify TS implements similarly.

## sql_literal.rb → sql-literal.ts — **DRIFT (necessary)**
